# DebateHelper - 辯論輔助工具

一個高效、本地運行的辯論輔助工具，旨在取代傳統的繁瑣流程，提供單機、高性能的筆記與稿件管理體驗。

## 專案核心特點

- **本地優先 (Local-first)**: 數據完全存儲於瀏覽器本地 (IndexedDB)，確保隱私、安全與極致的讀寫速度，無需依賴外部伺服器。
- **Markdown 支援**: 內建完整的 Markdown 渲染引擎，方便快速整理邏輯架構與重點。
- **Vim 模式支援**: 針對進階用戶與開發者，無縫整合 Vim 鍵位綁定，實現雙手不離鍵盤的高效編輯。
- **現代化設計**: 採用流暢的 UI 介面，支援深色模式 (Dark Mode) 與優雅的微動畫，提升長時間使用的舒適度。
- **數據導出與導入**: 支援將全部辯論數據匯出為 JSON 格式，方便備份與跨設備遷移。

## 技術棧

- **前端核心**: React 19 + Vite (提供極速的開發與建置體驗)
- **本地資料庫**: Dexie.js (強大且易用的 IndexedDB 封裝)
- **編輯器核心**: CodeMirror 6 + @uiw/react-codemirror
- **圖標系統**: Lucide React
- **樣式管理**: Vanilla CSS (基於 CSS 變數的現代化響應式設計)

### 核心模組深度解析

1.  **數據持久層 (`src/db.js`)**:
    使用 Dexie.js 定義 IndexedDB 資料表架構。主要包含 `contests` (辯題)、`notes` (筆記) 與 `scripts` (講稿) 三個資料表。負責處理所有的 CRUD 操作，確保數據的持久性與查詢效率。
2.  **狀態與佈局控制 (`src/App.jsx`)**:
    作為整個應用程式的神經中樞，管理當前選中的辯題 (`selectedContest`) 與當前操作的數據實體 (`editingData`)。負責協調 Sidebar、DataTable 與 DataEditor 之間的資料流與視圖切換。
3.  **導航系統 (`src/components/Sidebar.jsx`)**:
    提供直覺的側邊欄導航，允許用戶在不同的辯論主題間無縫切換，並提供快速新增辯題的操作入口。
4.  **編輯核心 (`src/components/DataEditor.jsx`)**:
    高度客製化的 CodeMirror 編輯器封裝。集成了 Markdown 語法高亮、GitHub Flavored Markdown (GFM) 支援，並可透過快捷鍵或按鈕動態開關 Vim 編輯模式。

## 如何開始

本專案提供兩種啟動方式：

如果您使用的是 Windows 系統，可直接雙擊專案根目錄下的 `run.bat`，它將自動為您啟動本地開發伺服器並在預設瀏覽器中開啟專案。

### 標準開發流程

1.  **安裝依賴套件**:
    ```bash
    npm install
    ```
2.  **啟動本地開發伺服器**:
    ```bash
    npm run dev
    ```
3.  **建置正式上線版本 (Production Build)**:
    ```bash
    npm run build
    ```
